Naming

CSS Class Naming

기본규칙

  • Kebab casing으로 영문 소문자, 숫자, hyphen(-)으로 조합하여 작성한다.
  • hyphen(-)조합은 최대 3단계를 초과하지 않는 걸 권장한다.
  • 단어와 숫자 조합은 하이픈(-)을 생략한다.
  • 숫자는 1부터 시작하며 '01' 과 같이 앞에 '0'을 붙여 사용하지 않는다.
  • Multi class의 조합은 3개 이하(우선순위 점수 33점 미만)을 권장한다.
  • id는 사용하지 않는다.

.base-main
.btn-mix-outlined-s
.list2
.btn-mix-outlined .on

Reserve a Prefix

예약된 접두사로 기본적으로 공통 사용하는 CSS naming 입니다.

reserve a prefix, Description
Reserve a Prefix Description
.base- 페이지내에서 유니크하게 하나만 사용하는 기본적인 구조로 주로 전체적인 레이아웃 요소나 페이지의 필수요소 이다.

base-wrap, base-header, base-body, base-content, base-footer, base-skip
.btn- button 관련된 스타일
.bul- bullet에 관련된 스타일
.wrap- 독립적인 item요소를 그룹으로 묶어주는 스타일

wrap-btn, wrap-bul
.form- 입력양식의 레이아웃 스타일
.inp- input, select, textarea에 관련된 스타일
.lb- radio,checkbox의 label 스타일
.ui- 자바스크립트 관련 클래스

Module

모듈단위로 구성하여 어느 곳에 위치하더라도 동일한 스타일을 제공하도록 구성한다.

Reserve a suffix Description
[module]-wrap 모듈단위의 최상위로 감싸는 영역
[module]-item 주 콘텐츠내용이 들어가는 영역(반복되는 영역) 

<div class="[module]">
	<div class="[module]-wrap>
		<div class="[module]-item>
			...
		</div>
		<div class="[module]-item>
			...
		</div>
	</div>
</div>
