웹 접근성

웹에서 제공하는 모든 전자정보를 모두(장애인, 고령자, 비장애인)에게 동등하게 접근할 수 있도록 보장을 하기 위해 2007년 장애인차별금지법 제정 후 시행되고 있는 법이다.

웹 접근성의 4가지 핵심원리

인식의 용이성
모든 콘텐츠는 사용자가 인식 할 수 있어야 한다.
운용의 용이성
사용자 인터페이스 구성 요소는 조작 가능하고 내비게이션할 수 있어야 한다.
이해의 용이성
콘텐츠는 이해할 수 있어야 한다.
견고성
웹 콘텐츠는 미래의 기술로도 접근할 수 있도록 견고하게 만들어야 한다.

title vs aria-label

title의 경우 스크린리더마다 설정에 따라 타이틀 텍스트 출력하는 방식이 달라진다. 특정 스크린리더에서는 화살표 키 이동 시 링크나 버튼의 요소의 추간된 타이틀 텍스틑 읽지 않는 경우도 있다.

aria-label 속성을 사용하게 되면 접근성 API에서 aria-label 속성으로 선언된 텍스트만 표시하므로 화면에 보이는 텍스트와 스크린리더용 텍스트를 함께 작성해 주어야 한다. 


//title
<button type="button"><span class="a11y-hidden">공지사항</span>더보기</button>

//aria-label
<button type="button" aria-label="공자사항 더보기">더보기</button>

명도 대비

저시력자, 고령자 등도 인식할 수 있도록 콘텐츠의 배경과 텍스트의 명도 대비를 4.5:1 이상으로 적용해야 한다.

명도 대비 예외 사항
텍스트가 18pt 또는 굵은(Bold) 14pt 이상일 경우 - 명도대비 3:1
확대 가능한 페이지의 일반 텍스트 또는 이미지 텍스트 - 명도대비 3:1

초점 이동

HTML 마크업을 논리적인 구조로 구성하여 키보드로 초점 이동 시 순차적인 순서로 이동 가능한게 한다.

  • 초점 이동 순서는 순환적이며, 반복적이어야 한다.
  • 모든 링크, 버튼, 입력창에는 초점이 이동 가능 해야한다..
  • 비활성화 요소에는 초점이 가지 않아야 하며, 숨겨진 콘텐츠에도 초점이 가지 않아야 한다.
  • 모달, 툴팁 등 화면을 가리는 레이어 팝업 요소로 포커스가 자동 이동한다.
  • 모달, 툴팁 등 화면을 가리는 레이어 팝업 요소가 닫히기 전까지는 해당 팝업 콘텐츠안에서 포커스를 순환시키며 닫는 경우에는 이전 포커스 위치로 이동한다.

조작 가능

미세한 조작이 어려운 장애나 환경에서도 사용자 입력 및 컨트롤 조작이 가능해야 한다.

  • 웹 페이지의 모든 컨트롤은 대각선 방향의 길이를 6mm 이상으로 제공한다.
  • 모바일 화면에서의 터치 오류의 최소화를 위해 컨트롤 크기는 가로와 세로가 9mm x 9mm 이상 되게 한다.
  • 링크, 사용자 입력, 기타 컨트롤 등이 안쪽 여백은 1px 이상의 여백을 두는 것이 좋다.
  • 모바일에서 컨트롤의 중심 간의 간격은 13mm x 13mm 이상이어야 사용자가 터치 오류를 범할 가능성이 낮아진다. (최소 9mm 권장)

사용자 요구에 따른 실행

사용자가 의도하지 않은 기능(새 창, 초점으로 인한 자동적 실행)은 예측할 수 없는 상황을 만들게 된다.

  • 드롭다운 메뉴나 입력창, 컨트롤(콤보박스, 라디오 박스, 체크박스 등)에는 선택 컨트롤과 함께 실행버튼을 제공해야 한다.
  • 사용자가 예측할 수 없는 상황에 새 창 또는 레이어 팝업 창을 통한 정보나 기능을 전달하면 안 된다.

제목 제공

페이지, 프레임, 콘텐츠 블록에는 적절한 제목을 제공해야 한다.

  • 모든 웹 페이지에는 해당 페이지를 간단명료하게 설명한 제목을 제공해야 하며 또한, 웹 페이지 제목은 서로 배타적이어야 한다.
  • 모든 웹 페이지의 프레임에는 각 프레임을 설명하는 간단명료한 제목을 제공해야 한다.
    아무런 내용이 없는 프레임에는 “빈 프레임” 등으로 제목을 제공한다.
  • 콘텐츠 블록에는 적절한 제목을 제공해야 한다.
  • 웹 페이지, 프레임 또는 콘텐츠 블록의 제목은 문장의 하나로 간주하여 불필요한 특수 기호를 반복하여 사용하지 않는다.

오류 정정

입력 오류가 발생 했을 경우 무엇이 잘못 되었는지 쉽게 찾아서 정정할 수 있는 방법을 제공해야 한다.

  • 오류가 발생한 위치와 오류를 유발하게 된 이유 등에 관한 정보를 알려 주어야 한다.
  • 시스템적인 오류는 해당되지 않는다.
